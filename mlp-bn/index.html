Overview — What Is This Notebook?

This notebook (build_makemore_batchnorm.ipynb) extends the basic MLP name-generator by adding Batch Normalization — a 2015 technique that dramatically stabilises training of deep networks. We also PyTorchify the code into reusable Linear, BatchNorm1d, and Tanh classes, then stack 6 layers deep.

What is Batch Normalization?
Batch Normalization normalises the inputs to each layer so they have zero mean and unit variance across a mini-batch, then applies learnable scale (γ) and shift (β) parameters. Introduced by Ioffe & Szegedy (2015).
Why do we need it?
Without BN, activations in deep networks explode or vanish — especially through saturating nonlinearities like tanh. BN keeps activations in a healthy range, stabilising gradients and speeding up convergence.

Cell 6: Embedding Matrix C

n_embd = 10   # each character → 10-dimensional vector
C = torch.randn((vocab_size, n_embd), generator=g)
# vocab_size=27, n_embd=10  →  shape: (27, 10)

What is C?

C is a lookup table — an embedding matrix. Every character index $i$ maps to a dense 10-dimensional vector:

$$e_i = C[i] \quad \in \mathbb{R}^{10}$$

This is much better than one-hot encoding (which would be 27-dimensional and sparse). The model learns what these 10 dimensions mean during training.

How does the lookup work?

# Xb has shape (batch_size, block_size) = (32, 3)
emb = C[Xb]         # shape: (32, 3, 10)
embcat = emb.view(emb.shape[0], -1)  # flatten → (32, 30)

Why embeddings instead of one-hot?
One-hot vectors have no notion of similarity. With embeddings, similar characters can end up close in 10-D space. Crucially, the 10-D space is much more compact — 27×10=270 parameters instead of 27×27=729 for a one-hot weight matrix.

Cell 6: First Linear Layer W₁ — Kaiming Init

n_hidden = 200
W1 = torch.randn((n_embd * block_size, n_hidden), generator=g) * (5/3)/((n_embd * block_size)**0.5)
# shape: (30, 200)
# scaling = (5/3) / sqrt(30) ≈ 0.304

The Forward Computation

$$h_{\text{pre}} = X \cdot W_1 \in \mathbb{R}^{B \times 200}$$

Where $X \in \mathbb{R}^{B \times 30}$ is the flattened embedding batch and $W_1 \in \mathbb{R}^{30 \times 200}$.

Why the Scaling Factor (5/3)/√n?

This is Kaiming (He) initialization adapted for tanh. The goal is to keep the variance of activations roughly constant as signals pass through the layer.

The Math

If $w \sim \mathcal{N}(0,1)$ and input $x$ has unit variance, then $h = xW$ has variance:

$$\text{Var}(h) = n_{\text{in}} \cdot \text{Var}(w)$$

To keep $\text{Var}(h) = 1$, we need $\text{Var}(w) = 1/n_{\text{in}}$, i.e., scale by $1/\sqrt{n_{\text{in}}}$.
The extra $5/3$ factor corrects for tanh's gain — tanh slightly compresses variance, so we pre-amplify.

Numerical Example: Why scaling matters

With $n_{\text{in}} = 30$, $B = 32$ samples:

①
No scaling: $W_1 \sim \mathcal{N}(0,1)$ → each neuron sums 30 random terms → std of $h_{\text{pre}} \approx \sqrt{30} \approx 5.5$ → tanh saturated
②
1/√n scaling: std of $h_{\text{pre}} \approx 1.0$ → some saturation still
③
(5/3)/√n scaling: std of $h_{\text{pre}} \approx 1.67$ → inputs fall in tanh's sensitive zone $(-2, +2)$
InitializationStd of h_preTanh SaturationGradient Health
No scaling~5.5~85% saturatedVery weak
1/√n (Xavier)~1.0~28% saturatedModerate

Cell 7: Batch Normalization — Full Math

# BatchNorm layer (inside training loop)
bnmeani = hpreact.mean(0, keepdim=True)    # μ — shape: (1, 200)
bnstdi  = hpreact.std(0, keepdim=True)     # σ — shape: (1, 200)
hpreact = bngain * (hpreact - bnmeani) / bnstdi + bnbias

The Three-Step Formula

1

Compute Batch Statistics

$$\mu = \frac{1}{m}\sum_{i=1}^{m} h_{\text{pre},i} \qquad \sigma^2 = \frac{1}{m}\sum_{i=1}^{m}(h_{\text{pre},i} - \mu)^2$$

Computed per feature (neuron), across all $m$ samples in the mini-batch. keepdim=True keeps shape (1, 200) for broadcasting.

2

Normalize (Z-score)

$$\hat{h}_i = \frac{h_{\text{pre},i} - \mu}{\sqrt{\sigma^2 + \epsilon}}$$

$\epsilon = 10^{-5}$ prevents division by zero when $\sigma^2 \approx 0$. After this step: mean≈0, variance≈1.

3

Scale and Shift (Affine Transform)

$$h_{\text{BN},i} = \gamma \hat{h}_i + \beta$$

$\gamma$ (bngain) and $\beta$ (bnbias) are learnable parameters. This lets the network un-do normalisation if needed, while still benefiting from the stable gradient flow.

Key Insight — Why Both Steps?
Step 2 alone would force every layer to always output mean=0, var=1 — too restrictive. The learnable γ and β in step 3 give back the network's expressiveness: it can choose to keep the normalised values OR re-scale/re-shift them to whatever range helps the task.

BN Parameters: γ (bngain) and β (bnbias)

bngain = torch.ones((1, n_hidden))   # γ — starts at 1 (identity scale)
bnbias = torch.zeros((1, n_hidden))  # β — starts at 0 (no shift)

What Each Parameter Does

ParameterSymbolInit ValueRoleEffect if Changed
bngainγ (gamma)1.0Scales normalised activationsγ>1 amplifies; γ<1 shrinks
bnbiasβ (beta)0.0Shifts normalised activationsMoves distribution up/down
bnmean_runningμ_run0.0Tracks global mean for inferenceNot a gradient parameter
bnstd_runningσ_run1.0Tracks global std for inferenceNot a gradient parameter

Why Start at γ=1, β=0?

This is the identity initialisation — at the start of training, BN outputs exactly the normalised values without any distortion. The network then learns non-trivial γ and β through gradient descent if needed.

What Happens With γ=2, β=1?

Say normalised activation $\hat{h} = [-1.0, 0.0, +1.0]$:

  • γ=1, β=0: $h_{\text{BN}} = [-1.0, 0.0, +1.0]$ → tanh outputs $[-0.76, 0.0, +0.76]$ → healthy gradients
  • γ=2, β=0: $h_{\text{BN}} = [-2.0, 0.0, +2.0]$ → tanh outputs $[-0.96, 0.0, +0.96]$ → more saturation
  • γ=1, β=1: $h_{\text{BN}} = [0.0, 1.0, 2.0]$ → tanh outputs $[0.0, +0.76, +0.96]$ → biased positive, reduced gradient

Running Statistics — Training vs. Inference

# Updated INSIDE training loop (no grad needed)
with torch.no_grad():
    bnmean_running = 0.999 * bnmean_running + 0.001 * bnmeani
    bnstd_running  = 0.999 * bnstd_running  + 0.001 * bnstdi

Exponential Moving Average (EMA) Formula

$$\mu_{\text{run}} \leftarrow (1 - \alpha)\,\mu_{\text{run}} + \alpha\,\mu_{\text{batch}} \qquad \alpha = 0.001$$
$$\sigma_{\text{run}} \leftarrow (1 - \alpha)\,\sigma_{\text{run}} + \alpha\,\sigma_{\text{batch}}$$

With $\alpha = 0.001$, the running stats move very slowly — it takes ~1000 steps to significantly shift. This gives a stable long-term estimate of the dataset-wide distribution.

Why Running Stats for Inference?

During training, BN uses the current batch's statistics — we have 32 samples, so the stats are well-defined. At inference time, we might predict one sample at a time. A batch of 1 has no meaningful variance. So we use the running averages collected during training instead.

Tanh Nonlinearity — The Saturation Problem

h = torch.tanh(hpreact)   # (B, 200) → values in (-1, +1)
$$h = \tanh(z) = \frac{e^z - e^{-z}}{e^z + e^{-z}} \qquad h \in (-1, +1)$$

Derivative — The Key to Gradient Health

$$\frac{d}{dz}\tanh(z) = 1 - \tanh^2(z)$$
Input ztanh(z)DerivativeGradient health
z = 00.0001.000Maximum — perfect
z = 0.50.4620.786Strong
z = 1.00.7620.420Moderate
z = 2.00.9640.071Weak — approaching saturation
z = 3.00.9950.010Near zero — vanishing
Saturation = Vanishing Gradients
When $|h| > 0.97$ (the code's threshold), the derivative is $< 0.06$ — less than 6% of the gradient survives. BN's job is to keep inputs $z$ close to 0, so tanh operates in its linear region where the derivative is near 1.

Output Layer & Cross-Entropy Loss

W2 = torch.randn((n_hidden, vocab_size), generator=g) * 0.01
b2 = torch.randn(vocab_size, generator=g) * 0

logits = h @ W2 + b2            # (B, 27) — raw scores
loss = F.cross_entropy(logits, Yb)  # scalar

Why W2 is Scaled by 0.01

At the start of training, we want the network to be uncertain about predictions — all 27 characters should be roughly equally likely. If W2 is large, some logits will be very high → very confident (wrong) predictions → high initial loss → unstable early training.

Cross-Entropy Loss Formula

$$\mathcal{L} = -\frac{1}{N}\sum_{b=1}^{N} \log \frac{e^{\text{logits}_{b,y_b}}}{\sum_j e^{\text{logits}_{b,j}}}$$

Where $y_b$ is the true next character index for example $b$. This penalises the model when the probability assigned to the correct character is low.

Why b2 is Initialised to Zero

Setting b2=0 means initial logits are near zero → softmax outputs near uniform 1/27 ≈ 0.037 → initial loss $\approx -\log(0.037) \approx 3.3$. The notebook shows the first loss printed as exactly 3.3239 — right on target.

Full Forward Pass — One Equation

Putting all steps together, the full forward pass from input characters to loss is:

$$\mathcal{L} = -\frac{1}{N}\sum_{b=1}^{N}\log \frac{\exp\!\left(\left[\tanh\!\left(\gamma\cdot\frac{X_b W_1 - \mu}{\sigma + \epsilon} + \beta\right)\right] W_2 + b_2\right)_{y_b}}{\sum_j \exp\!\left(\left[\tanh\!\left(\gamma\cdot\frac{X_b W_1 - \mu}{\sigma + \epsilon} + \beta\right)\right] W_2 + b_2\right)_j}$$
①

Embedding: $e_{b,t} = C[x_{b,t}]$

emb = C[Xb] — shape $(32, 3, 10)$

②

Concatenate: $X_b = \text{concat}(e_{b,1}, e_{b,2}, e_{b,3})$

embcat = emb.view(B, -1) — shape $(32, 30)$

③

Linear: $h_{\text{pre}} = X_b W_1$

hpreact = embcat @ W1 — shape $(32, 200)$

④

BatchNorm: $h_{\text{BN}} = \gamma \hat{h} + \beta$

Normalise, then scale+shift. Running stats updated here.

⑤

Tanh: $h = \tanh(h_{\text{BN}})$

Shape stays $(32, 200)$, values in $(-1, +1)$.

⑥

Output: $\text{logits} = h W_2 + b_2$

Shape $(32, 27)$ — one score per vocabulary token.

⑦

Loss: cross-entropy over logits

Scalar. Measures how well the model predicts the next character.

Backpropagation Through the Full Stack

for p in parameters:
    p.grad = None   # zero old gradients
loss.backward()    # compute all gradients via autograd

Gradient Flow — Layer by Layer

Step 1: Loss → Logits

$$\frac{\partial \mathcal{L}}{\partial \text{logits}_{b,j}} = p_{b,j} - \mathbf{1}[j = y_b]$$

Where $p_{b,j} = \text{softmax}(\text{logits}_b)_j$. This is the only "custom" gradient — PyTorch's F.cross_entropy handles it.

Step 2: Output Layer (W₂, b₂)

$$\frac{\partial \mathcal{L}}{\partial W_2} = h^T \delta_{\text{logits}} \qquad \frac{\partial \mathcal{L}}{\partial b_2} = \sum_b \delta_{\text{logits},b} \qquad \frac{\partial \mathcal{L}}{\partial h} = \delta_{\text{logits}} W_2^T$$

Step 3: Through Tanh

$$\frac{\partial \mathcal{L}}{\partial h_{\text{BN}}} = \frac{\partial \mathcal{L}}{\partial h} \cdot (1 - \tanh^2(h_{\text{BN}}))$$

This is the element-wise chain rule. Saturated neurons ($|h| \approx 1$) multiply by $\approx 0$, killing their gradient.

Step 4: BatchNorm Backward (the hard part)

$$\frac{\partial \mathcal{L}}{\partial \gamma} = \sum_i \delta_{h_{\text{BN}},i} \cdot \hat{h}_i \qquad \frac{\partial \mathcal{L}}{\partial \beta} = \sum_i \delta_{h_{\text{BN}},i}$$
$$\frac{\partial \mathcal{L}}{\partial h_{\text{pre},i}} = \frac{\gamma}{\sqrt{\sigma^2+\epsilon}} \left(\delta_i - \frac{1}{m}\sum_j \delta_j - \frac{\hat{h}_i}{m}\sum_j \delta_j \hat{h}_j\right)$$

Key insight: each sample's gradient depends on all other samples in the batch (through the $\sum_j$ terms). This coupling is why BN reduces internal covariate shift but also why small batches are problematic.

Step 5: Linear Layer (W₁)

$$\frac{\partial \mathcal{L}}{\partial W_1} = X^T \delta_{h_{\text{pre}}} \qquad \frac{\partial \mathcal{L}}{\partial X} = \delta_{h_{\text{pre}}} W_1^T$$

Step 6: Back to Embeddings (C)

The gradient w.r.t. $X$ scatters back into the rows of $C$ that were accessed for the current batch. Characters that appeared more often get more gradient updates.

Full Numerical Backward Example

Given: batch of 2, 2 hidden neurons

Forward values: $h_{\text{pre}} = [[2.0, 4.0], [1.0, 3.0]]$, $\mu=[1.5,3.5]$, $\sigma=0.5$, $\hat{h} = [[+1,+1],[-1,-1]]$

Suppose gradient from loss after tanh: $\delta_h = [[0.2, -0.1], [-0.2, 0.4]]$

Tanh derivative at $h = [0.76, 0.76, -0.76, -0.76]$: all $\approx 0.42$

So $\delta_{h_{\text{BN}}} = [[0.084, -0.042], [-0.084, 0.168]]$


Gradient w.r.t. γ: $\sum \delta \cdot \hat{h} = 0.084(1)+(-0.042)(1)+(-0.084)(-1)+0.168(-1) = -0.042$

Gradient w.r.t. β: $\sum \delta = 0.084-0.042-0.084+0.168 = 0.126$

After loss.backward(), each tensor's .grad holds the gradient of the loss w.r.t. that tensor. Plotting these shows how strong the learning signal is at each layer.

Why gradient histograms matter

Activation histograms show what's happening forward. Gradient histograms show what's happening backward. If gradients are tiny in early layers (std < 1e-4), weights there aren't learning — classic vanishing gradient problem. BN keeps gradient magnitude consistent across layers.

Saturation % → Gradient Magnitude Link

$$\frac{d}{dz}\tanh(z) = 1 - \tanh^2(z)$$

At $\tanh(z) = 0.97$: derivative $= 1 - 0.97^2 = 0.059$ — only 6% gradient passes through.

At $\tanh(z) = 0.99$: derivative $= 1 - 0.99^2 = 0.020$ — 2% passes through.

Saturation % of neuronsAverage gradient magnitudeLearning speed
~5% (with BN)~0.80Fast
~28% (Xavier init)~0.45Moderate

Why 0.97 as the saturation threshold?
$\tanh^{-1}(0.97) \approx 2.1$. At inputs beyond ±2, the derivative drops below 0.1 — a commonly accepted "effectively zero" gradient threshold. The notebook uses this empirical cut-off to report saturation statistics.

Cell 13: Custom Classes (PyTorchifying)

Linear Class

class Linear:
  def __init__(self, fan_in, fan_out, bias=True):
    self.weight = torch.randn((fan_in, fan_out), generator=g) / fan_in**0.5
    self.bias = torch.zeros(fan_out) if bias else None

  def __call__(self, x):
    self.out = x @ self.weight
    if self.bias is not None:
      self.out += self.bias
    return self.out

  def parameters(self):
    return [self.weight] + ([] if self.bias is None else [self.bias])

Forward math: $y = xW + b$. Weight init: scale by $1/\sqrt{n_{\text{in}}}$ to control variance (similar to Kaiming).

BatchNorm1d Class

class BatchNorm1d:
  def __init__(self, dim, eps=1e-5, momentum=0.1):
    self.gamma = torch.ones(dim)   # γ — learnable
    self.beta = torch.zeros(dim)   # β — learnable
    self.running_mean = torch.zeros(dim)  # not grad-tracked
    self.running_var = torch.ones(dim)

  def __call__(self, x):
    if self.training:
      xmean = x.mean(0, keepdim=True)
      xvar  = x.var(0, keepdim=True)
    else:
      xmean = self.running_mean
      xvar  = self.running_var
    xhat = (x - xmean) / torch.sqrt(xvar + self.eps)
    self.out = self.gamma * xhat + self.beta
    if self.training:
      with torch.no_grad():
        self.running_mean = (1-self.momentum)*self.running_mean + self.momentum*xmean
        self.running_var  = (1-self.momentum)*self.running_var  + self.momentum*xvar
    return self.out

Why bias=False with BatchNorm?
BatchNorm already has its own β (bias) parameter. If a Linear layer also has a bias b, then after normalisation that bias is subtracted out (it's absorbed into μ). So the two biases are redundant — removing Linear's bias saves parameters with no loss in expressiveness.

Special Initialisation for Output Layer

# make last BN layer less confident at init
layers[-1].gamma *= 0.1
# layers[-1].gamma is the BN gamma of the output layer

Shrinking the output layer's γ → logits start near zero → uniform softmax → lower initial loss → more stable early training. Same idea as initialising W2 with 0.01 scale.

Cell 22: BN Normalisation Widget (Math)

def normshow(x0):
  x = torch.randn(5) * 5
  x[0] = x0            # slider controls this value
  mu = x.mean()
  sig = x.std()
  y = (x - mu)/sig    # normalise

This widget shows how one extreme value ($x_0$) shifts the batch mean and how normalisation corrects for it.

Example: x = [10, −5, 0, 3, 2]

μ
$\mu = (10-5+0+3+2)/5 = 2.0$
σ
$\sigma = \sqrt{(64+49+4+1+0)/5} = \sqrt{23.6} \approx 4.86$
ŷ
$\hat{y} = [(10-2)/4.86, (-5-2)/4.86, \ldots] = [1.65, -1.44, -0.41, 0.21, 0.00]$
✓
$\text{mean}(\hat{y}) \approx 0$, $\text{std}(\hat{y}) = 1$ — normalised!

The blue dots (raw values) map to red dots (normalised). The blue curve is the original distribution; the red curve is the standard normal. Dragging $x_0$ to extremes shows how BN re-centres even skewed batches.

Cell 23–24: Linear vs Linear+BN Statistics

Cell 23 — Plain Linear Layer

c = b @ a      # 1000×1000 matrix × 1000 vector
# Results:
# c std: 31.0  (exploded!)
# a.grad std: 30.9  (exploded gradient!)
# b.grad std: 0.97  (reasonable)

Without any scaling, matrix multiplication with 1000 inputs causes std to grow by $\sqrt{1000} \approx 31.6$ — exactly what we see.

Cell 24 — Linear + BatchNorm

out = (x - xmean) / sqrt(xvar + 1e-5)
# Results:
# x std: 31.0  (still exploded going in)
# out std: 1.0  (BN fixes it!)
# x.grad std: 0.032  (compressed vs 0.97)
# inp.grad std: 1.0   (stable!)

BN catches the explosion: even though $x$ has std=31, BN normalises out back to std=1. Crucially, the input gradient stays at ~1 as well — BN also stabilises the backward pass.

Matrix Calculus: Gradient of a Matrix Multiply

$$c = b \cdot a \implies \frac{\partial \mathcal{L}}{\partial a} = b^T \cdot \frac{\partial \mathcal{L}}{\partial c} \qquad \frac{\partial \mathcal{L}}{\partial b} = \frac{\partial \mathcal{L}}{\partial c} \cdot a^T$$

2×2 Numeric Example

$a = [2, 1]^T$, $b = [[1,3],[2,4]]$, $c = b \cdot a = [5, 8]^T$

Gradient: $\delta_c = [1, -1]^T$

a.grad
$b^T \cdot \delta_c = [[1,2],[3,4]][1,-1]^T = [-1,-1]^T$
b.grad
$\delta_c \cdot a^T = [1,-1]^T [2,1] = [[2,1],[-2,-1]]$
